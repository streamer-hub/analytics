# Brand Voice

## 目的

このドキュメントは、StreamerHub の説明文・紹介文・投稿文・機能文言のトーンを統一するためのガイドです。

## ブランドの基本印象

StreamerHub は、以下の印象を目指します。

- 分かりやすい
- 実用的
- 見る楽しさを広げる
- オタク的な熱量はあるが、閉じすぎない
- データっぽさより「体験の便利さ」を先に伝える

## 基本トーン

### 1. 先に価値、後で機能

機能名を並べる前に、ユーザーが得することを先に言う。

- 良い例: 「複数視点をまとめて追いやすくする」
- 先に出しすぎない例: 「マルチビュー機能があります」

### 2. 難しい言葉より、使う場面で伝える

専門用語は必要最小限にし、具体的な利用場面をセットで示す。

- 良い例: 「イベント中に、今どの枠を見るべきか迷いにくくなる」

### 3. 熱量はあるが、誇張しすぎない

煽りすぎるより、便利さ・納得感・使う理由をはっきり示す。

### 4. ユーザーの視点で話す

「私たちが作った機能」より、「あなたがどう便利になるか」を優先する。

## 文体の指針

### 推奨

- 短めの文で区切る
- 1文1価値を意識する
- 具体例を入れる
- 読み手が利用シーンを想像できる形にする

### 非推奨

- 機能名の羅列だけで終わる
- 抽象的な形容詞だけで押す
- 誇張した断定を多用する
- 内輪向けの言い回しに寄りすぎる

## 訴求の優先順

1. 何が便利か
2. どんな場面で助かるか
3. なぜ他より追いやすいか
4. どんな機能で実現しているか

## コピーの方向性

### 強い軸

- 複数視点で追える
- 今見るべきものが見つかりやすい
- イベント全体を把握しやすい
- 推し視点から広げられる
- 話題の企画をすぐ追える

### 避けたい軸

- ただ「すごい」「最強」とだけ言う
- データ量だけを強みにする
- 視聴者の行動変化が伝わらない説明

## 文章タイプ別ガイド

### サービス紹介

- 価値を短く一発で伝える
- 初見でも分かる言葉を優先する
- 機能説明は2段目以降に置く

### X 投稿

- 冒頭で利用場面を出す
- 1投稿1テーマに絞る
- CTA は明確にする
- 体験イメージが湧く表現を優先する

### 機能説明

- 何ができるか
- いつ便利か
- どう使うか
の順で書く

### 仕様書内の説明文

- 感情語よりも、意図と効果を明確にする
- ただしユーザー価値を消さない

## 例文

### 良い例

- 「イベント中に、複数の配信視点をまとめて追いやすくします。」
- 「推しの枠を見ながら、関連する視点もすぐ追加できます。」
- 「話題の企画を、どこから見ればいいか迷いにくくします。」

### 弱い例

- 「高機能な新時代の配信体験を提供します。」
- 「多数のデータを統合した革新的なプラットフォームです。」
- 「業界最高峰のUXです。」

## トーンの最終判断基準

迷ったときは、以下を満たしているかで判断します。

- 読んだ瞬間に使う場面が想像できるか
- 初見ユーザーでも意味が通るか
- 機能ではなく価値が主語になっているか
- 誇張より納得感があるか
