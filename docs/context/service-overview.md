# Service Overview

## サービス名

StreamerHub

## 一文定義

StreamerHub は、配信者・イベント・動画・同時視聴の文脈を整理し、視聴者が「今どこを見るべきか」を見つけやすくするための視聴支援プラットフォームです。

## 解決したい不満

配信視聴には、次のような不満が発生しやすいです。

- コラボや大型企画で、誰がどこで何をしているか把握しにくい
- 複数視点で見たほうが面白いのに、タブや配信枠の管理が面倒
- 企画の流れを追えず、途中参加だと何が起きているのか分かりづらい
- 特定イベントに関連する配信者や視点が散らばっていて、発見しづらい
- 「面白い瞬間」を見るまでの導線が長い

## 提供する価値

StreamerHub が提供する主な価値は以下です。

1. **発見性の向上**
   - 今追うべき配信・視点・関連コンテンツが見つかりやすい
2. **比較視聴のしやすさ**
   - 複数視点をまとめて見られる
3. **文脈理解の補助**
   - イベント単位・グループ単位で状況を追いやすい
4. **視聴行動の短縮**
   - 「見たいものに辿り着くまでの操作」を減らす

## 想定ユーザー

### 主対象

- VTuber / ストリーマー / 配信イベントを追う視聴者
- 複数視点で配信を同時に見たいユーザー
- イベントを俯瞰して追いたいユーザー
- 特定の配信者だけでなく、企画全体を楽しみたいユーザー

### 副対象

- 配信者の動きやイベント構造を整理して見たいリサーチ目的のユーザー
- 話題のイベントを追い始めた新規層
- 配信者・切り抜き視聴者・SNS経由で流入したライト層

## 現在の中核機能

### 1. マルチビュー

- 複数の配信視点を並べて見られる
- 配信者単位ではなく「企画や視聴体験」基準で整理しやすい
- 同じ出来事を複数枠から比較できる

### 2. イベント単位の整理

- 企画 / イベントごとに関連配信や視点をまとめる
- 特定イベントの追跡がしやすい
- 視聴者が「今どの枠を見るべきか」を判断しやすい

### 3. 配信者・動画の発見支援

- 関連する配信者、動画、文脈をまとめて見せる
- ただの一覧ではなく、視聴意図に沿って回遊しやすくする

### 4. 将来的に強化したい領域

- 自動タグ付け
- 文脈ベースのおすすめ
- コラボ判定
- ランキング / 伸びの可視化
- イベントごとの動向整理

## StreamerHub の強み

### 強み1: 視聴体験起点

単純な配信一覧やチャンネル一覧ではなく、「どう見たいか」「何を追いたいか」を起点に設計できることが強みです。

### 強み2: イベント文脈に強い

大規模企画や複数人参加型の配信で価値が出やすく、単独視聴では取りこぼす情報を補完できます。

### 強み3: 情報整理と発見の中間にある

ただのデータベースでも、ただのプレイヤーでもなく、「見つける」と「見る」をつなぐ導線に位置している点が特徴です。

## 今後の方向性

- イベント視聴の最適導線をさらに短くする
- 新規ユーザーが価値をすぐ理解できるようにする
- SNS / 検索 / 紹介動画からの流入に対応する
- 視聴体験と情報整理を一体化する
- 機能ごとの価値を分かりやすく見せる

## 提案時に守る前提

新機能や施策を考えるときは、以下を優先します。

1. 視聴者が「今見るべきもの」に早く到達できるか
2. 既存のマルチビュー体験を強化できるか
3. イベント・企画文脈の理解を深められるか
4. ライト層にも価値が伝わるか
5. 実装コストに対して継続利用価値があるか
