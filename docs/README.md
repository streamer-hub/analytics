# StreamerHub Docs

この `docs/` ディレクトリは、Claude Code に StreamerHub の前提知識・判断基準・出力形式を安定して渡すための共通ドキュメント群です。

## 目的

- 毎回同じ背景説明をしなくても、一定品質で機能案・仕様・マーケティング案を出せるようにする
- 機能提案・コンテンツ制作・分析・振り返りを同じ前提で進められるようにする
- 生成物の粒度や評価基準を固定し、案のブレを減らす

## ディレクトリ構成

- `context/`: サービスの基本情報、用語、対象ユーザー、ブランドトーン
- `strategy/`: 中長期の優先順位、KPI設計、守るべき制約
- `marketing/`: 集客チャネル、訴求軸、投稿設計、ハッシュタグ運用
- `competitors/`: 主要競合の整理と自サービスとの差分
- `specs/`: 仕様テンプレートと提案仕様のドラフト置き場
- `analytics/`: 指標定義、ダッシュボード観点、イベント施策の測定方法

## Claude Code での想定用途

- 新機能のアイデア出し
- 機能案の仕様化
- X / Shorts / 記事向けの訴求作成
- イベント連動の施策設計
- 施策の事後振り返り

## 運用ルール

1. 新しい判断基準が固まったら、まず `docs/` を更新する
2. 一時的なメモは直接散らさず、該当カテゴリに統合する
3. 機能仕様は `specs/proposed/`、テンプレートは `specs/templates/` に分ける
4. 数値指標の定義変更は、必ず `analytics/metric-definitions.md` も更新する

## 更新優先度

更新頻度が高い順に見ると、基本的に以下です。

1. `marketing/`
2. `strategy/`
3. `analytics/`
4. `specs/`
5. `context/`
6. `competitors/`
