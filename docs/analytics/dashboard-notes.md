# Dashboard Notes

## 目的

このドキュメントは、StreamerHub のダッシュボードで何をどう見るべきかを整理するためのメモです。

## 基本方針

ダッシュボードは「数字を並べる場所」ではなく、「次に何を改善するかを判断する場所」として設計します。

## 最低限見るべきビュー

### 1. サービス全体ビュー

見る項目:
- 総セッション数
- 有効セッション数
- 価値体験セッション数
- 再訪率

目的:
- 全体の健康状態を見る

---

### 2. 流入チャネルビュー

見る項目:
- X 流入
- 検索流入
- 直接流入
- シェア経由流入
- 各チャネルの価値体験率

目的:
- どの流入が質が高いかを判断する

---

### 3. 主要導線ビュー

見る項目:
- イベントページ表示数
- イベントページからの主要行動率
- 配信者ページからの関連視点追加率
- マルチビュー起動率

目的:
- 導線上の詰まりを見つける

---

### 4. イベント別ビュー

見る項目:
- イベントページ流入
- マルチビュー起動数
- セッション時間
- 再訪率
- イベント中の投稿経由流入

目的:
- どのイベントで価値が最も出たかを見る

---

### 5. コンテンツ / 訴求ビュー

見る項目:
- 投稿別クリック率
- 投稿別コア価値到達率
- 画像 / 動画 / テキストの差

目的:
- どの訴求が強いかを見る

## 見る順番

1. 全体が悪化していないか
2. 流入が悪いのか、導線が悪いのか
3. どのページで落ちているか
4. どの投稿 / 施策が効いたか
5. 次の改善対象を決める

## 異常検知の観点

### 流入増・価値体験減

訴求は届いているが、期待と体験がズレている可能性がある。

### イベント流入増・起動率低下

イベントページや導線が弱い可能性がある。

### 投稿クリック率高・再訪低

興味は引けているが、コア価値や継続価値が弱い可能性がある。

### 全体セッション維持・再訪低下

新規は入っていても、定着理由が不足している可能性がある。

## ダッシュボード設計ルール

- まず全体、次に分解
- 指標は目的別にまとめる
- 「良かった / 悪かった」だけでなく、次の打ち手が見える配置にする
- イベント時と平時は分けて表示する
