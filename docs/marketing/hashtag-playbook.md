# Hashtag Playbook

## 目的

このドキュメントは、StreamerHub の X 投稿でハッシュタグをどう使うかの基本方針を整理するものです。

## 基本方針

ハッシュタグは「見つけてもらう補助」であり、主役ではありません。

最優先は以下です。

1. 投稿本文の訴求
2. 話題との接続
3. 導線の明確さ
4. 必要なタグの選定

## ハッシュタグの役割

- イベント文脈に接続する
- 話題検索で見つけてもらう
- 投稿テーマを明示する

## 使用カテゴリ

### 1. イベント名タグ

例:
- `#NEWTOWN`
- 大型企画の公式 / 慣用タグ

#### 用途

- 話題への接続
- イベント参加者・視聴者への露出

### 2. サービス固有タグ

例:
- `#StreamerHub`
- `#StreamerHubJP`

#### 用途

- 投稿の蓄積
- 自分たちの発信整理
- 後から追いやすくする

### 3. 周辺カテゴリタグ

例:
- 配信文化、VTuber、イベント関連など

#### 用途

- 必要な場合のみ補助的に使用
- つけすぎない

## ルール

### 基本数

- 1〜3 個を基本にする
- 多くても 4 個まで

### 優先順位

1. イベント名タグ
2. サービス固有タグ
3. 補助タグ

### 避けること

- 関係の薄い人気タグを混ぜる
- タグだらけで本文の可読性を下げる
- 毎回同じ構成で機械的に付ける

## 投稿タイプ別の使い分け

### イベント連動投稿

- イベント名タグ: 必須寄り
- サービスタグ: 追加
- 補助タグ: なくてもよい

### 通常の価値訴求投稿

- サービスタグ中心
- イベント名タグは無理につけない

### 使い方紹介

- 基本はサービスタグのみ
- タグ数を増やしすぎない

## 推奨テンプレ

### イベント投稿

- 本文: 今回みたいな複数視点イベント、まとめて追いやすくしたい人向け
- タグ: `#NEWTOWN #StreamerHubJP`

### 通常投稿

- 本文: 複数視点で配信を追いたいときに、今見るべき枠へ行きやすくする
- タグ: `#StreamerHubJP`

## ハッシュタグ見直しルール

- イベントが終わったら、期間依存タグは引きずらない
- 投稿の反応を見て、使うタグパターンを定期的に更新する
- 反応が悪い場合は、タグより本文と訴求を先に見直す
