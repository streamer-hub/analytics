# ツール追加提案: 新規獲得を強くするツール

> 作成日: 2026-03-03
> テーマ: `新規獲得を強くするツール`
> 目的: X中心の現在の流入構造を維持しつつ、ツイート→新規ユーザーの変換率を上げる

---

## 前提: 現状のボトルネック

分析データ（2026-03-03版 / 35投稿 / GAデータ 468ユーザー）から抽出した課題。

| ボトルネック | 数値 | 意味 |
|---|---|---|
| MultiView直帰率 | 41.5% | 来た人の約4割が何も得ずに帰る |
| UTM未付与 | 全投稿0件 | ツイートごとの新規獲得数が計測できない |
| X流入依存率 | 81.0% | X以外からの新規獲得がほぼゼロ |
| viral投稿 | 0件 | フォロワー以外には届いていない |
| 開始直後ツイートの人力依存 | タイミングが毎回ブレる | 最も効くフェーズ（avg 34 clicks）を取りこぼしている |

## 候補ツール一覧（7候補）

---

### 🔴 ツール1: UTMパラメータ自動付与ツール
**ユーザー向け / 内部運用向け**: 内部運用向け

**何ができるか**

ツイート投稿時に貼るURLへ自動でUTMパラメータを付与する。
たとえば `https://player.streamerhub.jp/newtown` → `https://player.streamerhub.jp/newtown?utm_source=twitter&utm_campaign=newtown_20260303_01` に変換。

**どんな場面で効くか**

毎回ツイートのURLを貼るとき。今は毎回手動コピペのため UTM がゼロ件。1件もUTMがない状態では「このツイートが何人の新規ユーザーを獲得したか」がGAで追えない。

**どこに効くか**: 新規獲得の測定基盤（他のすべての改善の前提になる）

**MVP**

- ツイート作成スキル（newtown-multiview-tweet など）の内部で、URLを発行する際に UTM パラメータを自動付与するルールを組み込む
- `?utm_source=twitter&utm_campaign=newtown_[YYYYMMDD]_[投稿番号]` を自動生成
- 追加作業: ゼロ（ツイート生成の内部処理として実装するだけ）

**必要なデータ/連携**: なし（URL文字列の置換のみ）

**継続運用コスト**: ほぼゼロ（スキルに組み込むだけ）

**優先度**: ★★★★★（最優先・今すぐ）

> **理由**: UTM がないと他の全改善の効果測定ができない。他ツールより先に入れないと投資対効果が測れないまま積み上がってしまう。実装コストは最も低い。

---

### 🔴 ツール2: MultiViewライブ状態リアルタイム表示ツール
**ユーザー向け / 内部運用向け**: ユーザー向け

**何ができるか**

StreamerHubのMultiViewページに、配信者が「現在配信中🔴」または「オフライン⚫」かをリアルタイムで表示する。ツイートを見てサイトに来たときに配信が終わっていても、次回配信予定がわかる。

**どんな場面で効くか**

「パレト銀行強盗が始まる👀」というツイートを見て1時間後にクリックしたユーザーが「あれ、もう終わってる」と気づいて即離脱している（直帰率41.5%の主因と推定）。リアルタイムのオンライン/オフライン表示があるだけで「今見られるかどうか」がわかり、無駄な離脱が減る。

**どこに効くか**: 新規獲得（流入したユーザーの定着率向上 / 直帰率削減）

**MVP**

- Twitch API / YouTube Live API で配信中かどうかをチェックし、MultiViewページのバナーに「🔴 現在配信中」または「次回: ○日○時の予定」を表示する
- ページロード時 or 30秒ごとにAPI問い合わせ
- 配信終了後は「今日の見逃しハイライトはこちら」などのCTAへ切り替え

**必要なデータ/連携**: Twitch API または YouTube Data API（配信ステータス取得）

**継続運用コスト**: API呼び出しコスト（Twitchは無料枠内が多い）、配信スケジュールの事前登録が必要

**優先度**: ★★★★★（最優先・今すぐ）

> **理由**: 直帰率41.5%の削減が最大のROI。流入数を増やすより「来た人を逃さない」方が即効性が高い。X流入81%という構造ではここを1%改善するだけで新規獲得数が直接増える。

---

### 🟠 ツール3: 配信開始自動検知 × 即時ツイート下書き生成ツール
**ユーザー向け / 内部運用向け**: 内部運用向け

**何ができるか**

配信者がTwitch/YouTubeで配信を開始したことを自動検知し、3分以内に最適なツイート下書きをSlackやメール、または専用UIに通知する。「今から開始直後ツイートを出すべき」というアラートを受け取り、承認ボタン1つでXに投稿できる。

**どんな場面で効くか**

データ上「開始直後（配信開始から3〜5分以内）」がURL Clicks 34.0と全フェーズ最高。しかし人力で毎回タイミングを合わせるのは難しい。自動検知があれば最大効率のタイミングを逃さなくなる。

**どこに効くか**: 新規獲得（流入量の上限を引き上げる）

**MVP**

1. 対象配信者（NEWTOWN ギャング）のTwitchチャンネルをリスト管理
2. Webhookまたはポーリング（5分ごと）で配信開始を検知
3. 検知したら `newtown-multiview-tweet` スキルを自動起動し、最適フック・タグを含む下書きを生成
4. Slack通知 or メールで「今すぐ投稿できます」とリンク付きで送付
5. 担当者がワンクリックで承認・投稿（またはX APIで自動投稿）

**必要なデータ/連携**: Twitch EventSub API（配信開始Webhook）、Slack API（通知）、Twitter API v2（自動投稿オプション）

**継続運用コスト**: Twitchは無料枠あり。Twitter API Basic Plan（$100/月）が必要な場合あり。自動投稿まで行くなら月額コスト発生。

**優先度**: ★★★★☆（次に検討）

> **理由**: 流入量の「天井」を上げる。手動運用では人が見ていないと開始直後の投稿を逃し、その日のURL Clicksが大幅低下する。ツール1・2の後に入れると、「来た人を逃さない」→「そもそも来る数を増やす」の順序になる。

### 🟠 ツール4: ギャング別ページ誘導リンク管理ツール
**ユーザー向け / 内部運用向け**: 内部運用向け

**何ができるか**

ツイートのリンク先URLを、MultiView固定から「ギャング別ページ」や「警察ページ」など内容に合ったページに動的に切り替えるルールを管理する。ツイート生成スキル内に「この投稿がDear13roなら → Dear13roページURL、NEWTOWN全体なら → MultiViewURL」というルールを組み込む。

**どんな場面で効くか**

GAデータで、Dear13roギャングページの直帰率は26.4%（MultiViewの41.5%より15pt低い）。警察ページは14.3%と最低。同じ流入数でも直帰率が15pt下がれば新規ユーザーの定着数が相当増える。

**どこに効くか**: 新規獲得（流入後の定着率向上）

**MVP**

ツイート生成スキル内で、投稿対象のギャングに応じてURLを自動選択するルールを実装。
- Dear13ro投稿 → `/newtown/dear13ro` ページURL
- AtroceMyrtille投稿 → `/newtown/atroce-myrtille` ページURL
- NEWTOWN全体投稿 → MultiViewURL（現状維持）

**必要なデータ/連携**: StreamerHub内の各ギャングページURL一覧（静的リスト）

**継続運用コスト**: 事実上ゼロ（スキル内のルール更新のみ）

**優先度**: ★★★☆☆（次に検討）

> **理由**: コストがほぼゼロで直帰率を下げられる。ツイート生成スキルへの数行の変更で実現できる。ただし、ページ側のコンテンツ品質が前提になるため、ページが十分に整備されてから実施が望ましい。

### 🟡 ツール5: 拡散型投稿（問いかけ型）生成ツール
**ユーザー向け / 内部運用向け**: 内部運用向け

**何ができるか**

「今夜のNEWTOWN、どのギャングから追う？」「先週の最高のシーン、見た？」のような返信・リポストを引き出す問いかけ投稿を生成する専用モードをツイート生成スキルに追加。For You 配信を拡大し、フォロワー以外の新規ユーザーへリーチする。

**どんな場面で効くか**

現在35投稿すべてがURL Clicks特化型またはlow_performerであり、viral_focused（拡散型）投稿は0件。Xアルゴリズム上、返信が発生すると「ネットワーク文脈」が生まれFor You露出が拡大する。フォロワー外の新規ユーザーに届く唯一の手段として機能しうる。

**どこに効くか**: 新規獲得（フォロワー外へのリーチ拡大 → 潜在新規ユーザーの流入）

**MVP**

ツイート生成スキルに `mode: viral` オプションを追加。
- イベント前日〜当日朝に「どのギャングを応援する？」「先週一番熱かったシーンはどれ？」型の投稿案を3パターン生成
- URLなし、ギャング名を選択肢として明記
- エンゲージメント計測: 返信数・リポスト数をx_post管理ファイルに記録欄追加

**必要なデータ/連携**: 現行の patterns.md（フック・ギャング名情報）

**継続運用コスト**: 週1〜2本の問いかけ投稿コスト（運用工数：10〜15分/週）

**優先度**: ★★★☆☆（次に検討）

> **理由**: 「現在の35投稿がすべてURL Click型」という構造的な問題を解消する。ただし検証には時間がかかるため、ツール1〜2の後、測定基盤が整ってから試すのが望ましい。

### 🟢 ツール6: SEOコンテンツ自動生成ツール
**ユーザー向け / 内部運用向け**: 内部運用向け

**何ができるか**

NEWTOWNのイベント後に「NEWTOWN パレト銀行強盗 まとめ」「Dear13ro マルチビュー 応援方法」などのキーワードを狙ったショートブログ記事を自動生成する。配信記録 + ツイートデータをもとに記事構成を生成し、StreamerHub公式ブログ/Noteへの掲載候補を出力する。

**どんな場面で効くか**

GAでYahoo/Google検索流入が3.7%（40セッション）と、少ないながら増加傾向。今は検索して来た人が偶然サイトに辿り着いているが、SEOコンテンツを出せば「NEWTOWN 見方 マルチビュー」などで検索する層を能動的に取り込める。

**どこに効くか**: 長期の新規獲得（X一極依存からの脱却）

**MVP**

- イベント終了後に投稿データ・ギャング情報をインプットとして、500〜800字のブログ記事草稿を自動生成
- 対象キーワード: 「NEWTOWN [強盗名] マルチビュー」「[ギャング名] 配信 まとめ」
- 出力先: `output/seo/[YYYYMMDD]-[topic].md` に保存し、人間がチェックして掲載

**必要なデータ/連携**: イベント記録データ、ギャング情報、StreamerHub URL

**継続運用コスト**: 記事の確認・掲載に週30〜60分。ただし効果が出るまで3〜6ヶ月かかる。

**優先度**: ★★☆☆☆（将来の大型投資）

> **理由**: 長期的な「X依存リスク分散」として価値が高いが、即効性がない。ツール1〜3の測定基盤と流入最大化が済んでから着手すべき。

### 🟢 ツール7: クロスプラットフォーム拡散ツール（TikTok/YouTube Shorts）
**ユーザー向け / 内部運用向け**: 内部運用向け

**何ができるか**

NEWTOWNの名場面クリップ（配信のハイライト）を15〜60秒のショート動画に加工してTikTok・YouTube Shortsへ投稿する補助ツール。「TikTokで見て初めてNEWTOWNを知った」新規ユーザー層を開拓する。

**どんな場面で効くか**

X以外の流入がほぼゼロの状態を変え、Xを見ていない層（TikTokユーザー・YouTube視聴者）へリーチする。Xのフォロワーが頭打ちになった段階での成長チャネル。

**どこに効くか**: 新規獲得（まったく新しいユーザー層のリーチ）

**MVP**

- 配信者がクリップ切り抜きURLを指定 → FFmpegなどで字幕付きショート動画に変換
- 「この強盗、マジかww #NEWTOWN #streamerhub」のような文面を自動生成して投稿案を出力
- TikTok Draft APIまたはYouTube Data APIで下書き保存

**必要なデータ/連携**: 配信アーカイブ（Twitch Clip URL）、TikTok API、YouTube Data API

**継続運用コスト**: クリップ選定に人手が必要。API連携のメンテナンスコスト。コンテンツ権利の確認必須。

**優先度**: ★★☆☆☆（将来の大型投資）

> **理由**: 可能性は大きいが、実装・運用コストが最も高い。著作権（配信者との合意）とAPIコストの問題もあり、他ツールで基盤を整えてから着手する。

## 最初の1つ: ツール1（UTMパラメータ自動付与）

**なぜこれか**

すべての改善活動の「計測基盤」がない状態。ツール2〜7のどれを入れても「どのツイートが何人の新規ユーザーを獲得したか」がわからないまま続けることになる。ツイート生成スキルへの小さな修正で実現でき、即日導入できる。コストゼロ。

**具体的な最初のアクション**

`newtown-multiview-tweet` スキルのURL出力部分に、以下を自動付与するルールを追加する:
```
?utm_source=twitter&utm_campaign=newtown_[YYYYMMDD]_[投稿番号]
```

## その次の1つ: ツール2（MultiViewライブ状態リアルタイム表示）

**なぜこれか**

UTMが入ったことで「どのツイートから何人来た」がわかるようになる。次は「来た人の41.5%が即離脱している問題」を解消する番。Twitch APIで配信状態を取得してページに「🔴 配信中」を表示するだけで、無駄な離脱を減らして新規ユーザーの定着率が上がる。StreamerHubの開発リソースが必要になるが、影響が最も大きい改善。

**具体的な最初のアクション**

StreamerHubのMultiViewページに「現在配信中 / オフライン」のバナーコンポーネントを実装する。対象配信者のTwitchチャンネル名はすでにデータとして持っているため、API連携だけ追加すれば動く。

---

## 付記: 今すぐできる無コスト改善

スキル改修や開発を待たず、運用ルールとして即日実行できる改善。

1. **ツイートURLへの手動UTM付与** — ツール1の正式実装前に、人力でURLに `?utm_source=twitter` を追加するルールを徹底する
2. **ギャング別ページへのリンク変更** — Dear13ro投稿は今すぐMultiViewではなくDear13roギャングページをリンクする（スキル改修不要、運用変更だけで可能）
3. **配信終了後のツイート停止ルール** — 配信が終わった後にURL付きのツイートを出さない。オフライン時はURLなしの問いかけ投稿に切り替える
