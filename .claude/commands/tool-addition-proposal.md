# /tool-addition-proposal

StreamerHub に追加する「次のツール」を提案するためのコマンド。  
ここでいうツールは、以下のどちらでもよい。

- **ユーザー向けツール**
  - 例: 配信比較、通知、タグ補助、見逃し防止、切り抜き補助、イベント追跡
- **内部運用ツール**
  - 例: マーケ分析ダッシュボード、投稿補助、競合監視、ランキング自動生成、運用自動化

## 目的
- 「何を追加すべきか」を優先順位つきで明確化する
- 追加ツールが新規獲得・継続利用・運用効率のどこに効くか整理する
- 実装前に、導入価値と運用コストのバランスを見る

## 推奨サブエージェント
1. `market-insight-searcher`
2. `marketing-strategy-agent`
3. `technical-marketing-agent`
4. `social-growth-hacker`
5. `legal-compliance-guard`（必要時）

## 実行手順
1. `$ARGUMENTS` からテーマを確定する
   - 例: `新規獲得を強くするツール`, `配信者向け便利機能`, `内部運用自動化`
   - 空なら「何を強化したいか」を1つだけ確認する
2. 先に `ideas/[YYYYMMDD]-[topic].md` を作り、仮説を保存する
3. 以下の2軸で候補を考える
   - **ユーザー価値**（使いたくなるか / 続けたくなるか）
   - **運用価値**（作業が減るか / 速度が上がるか / 再利用できるか）
4. 最低5候補出す
   - うち少なくとも2つは「すぐ着手できる小さめ」
   - うち少なくとも1つは「将来の差別化になる大きめ」
5. 各候補について整理する
   - ツール名
   - ユーザー向け / 内部向け
   - 何ができるか
   - どんな場面で効くか
   - 新規獲得 / 継続 / 運用効率 のどれに効くか
   - MVP
   - 必要そうなデータ/連携
   - 継続運用コスト
   - 優先度
6. 候補を優先順位順に並べる
   - **今すぐやる**
   - **次に検討**
   - **後で大型投資**
7. 最後に「最初の1つ」と「その次の1つ」を選び、理由を書く
8. 最終版を以下に保存する
   - `output/specs/tool-additions/[YYYY-MM-DD]-[topic].md`

## 出力フォーマット
`templates/tool-addition-proposal.md` を基本テンプレにする。

## 出力の質ルール
- 単に「便利」ではなく、**何がどれだけ良くなるか**を書く
- 追加後の利用イメージ（導線）を具体化する
- API・自動化・データ連携が絡む場合は、難しい言い方を避けて価値に翻訳する
- 小さい候補と大きい候補を混ぜて、選べるようにする
