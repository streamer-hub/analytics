### 解釈（Hook Analyst + X Algorithm観点）

**1位: が始まる/始まった（平均URL Clicks 34.0 / 平均CTR 1.80% / 平均Imp 1,928）**

「始まる」という動詞が持つ開始の瞬間性が、読み手に「今見なければ乗り遅れる」という緊迫感（FOMO）を与える。Xアルゴリズムのvelocityシグナル（投稿後数分〜5分前後の初期反応の密度）がFor Youへの候補昇格に直接影響するため、イベント開始から0〜3分以内の投稿が必須条件となる。代表例：「パレト銀行強盗が始まる👀 URLから同時視聴してみよう！」（55 Clicks / CTR 2.22%）。同フック内でも明示的CTAがある投稿（55 clicks）とCTAが弱い投稿（19 clicks）で約3倍の差があり、CTA省略は避けること。

**2位: サービス紹介（平均URL Clicks 25.6 / 平均CTR 2.34% / 平均Imp 949）**

インプレッション949と低いにもかかわらずCTRが全フック中最高という特異な構造を持つ。ハッシュタグをフックの冒頭に置くパターン（`#Dear13ro マルチビューを使って〜`）がCTR 2.34%を記録。ギャング固有タグが視覚的に「このツイートは誰向けか」を瞬時に伝えるセグメント機能を果たし、ファンが自己同一視しやすい。「応援しよう」という行動提案がクリック行為と応援行為を等価にしている。代表例：`#Dear13ro マルチビューで応援しよう！`（63 Clicks / CTR 2.57%）、`#SANTACLAWS マルチビュー〜`（47 Clicks / CTR 3.91%）。

**3位: その他（平均URL Clicks 18.4 / 平均CTR 1.26% / 平均Imp 1,292）**

投稿数14件と最多カテゴリ。「Dear13roがメイズバンク強盗を始めたみたい」（64 Clicks・全投稿中最高）のように「みたい」という伝聞表現が知的好奇心を刺激するパターンや、「ついに」という副詞が積み重なった文脈を知るコアファンに強く響くパターンが含まれる。内容の質によって個別バラつきが大きいため、平均値より個別事例の型を参考にすること。

**下位フックの判断**

- **修正・報告（廃止推奨）**: CTR 0.04%は全体平均の約30分の1。Imp 1,298と一定数あるにもかかわらずクリック率がほぼゼロ。「[修正報告]」形式はSNSのエンタメ文脈において「停止」「問題発生」の印象を与える。報告が必要な場合はリプライ限定またはベネフィット訴求に転換する。
- **感情先行（廃止推奨）**: 情報量ゼロの感情語はImp 622（全フック最低）を記録。「傑作」のような感情評価語のみを冒頭に置くフックは、何についての「傑作」なのかが瞬時に伝わらない。感情語は「固有名詞 + 感情語」の順序に変える。
- **時刻告知・質問形（改変推奨）**: Impは1,760・1,776と比較的高いがCTRは0.34%に留まる。時刻告知は単体では弱く「が始まる/始まった」型と組み合わせて「〇〇時から強盗が始まる」の複合フックに統合するのが現実的。質問形は内輪ネタ質問からURL誘導ではなく返信誘発用途に限定する。

### 転用できるフックテンプレート

1. **開始宣言型（緊迫感フック）**
   - 型: `[強盗/イベント名]が始まる👀 URLから[行動語]してみよう！`
   - 推奨フェーズ: 開始直後（ミッション開始から5分以内）

   - 根拠: 「が始まる/始まった」フックの平均URL Clicks 34.0は全タイプ最高値。開始の瞬間を告げることで視聴乗り遅れへの焦りを生む。

2. **ギャング名ハッシュタグ先頭+応援促進型**
   - 型: `#[ギャング名] マルチビューを使って[ギャング名]を応援しよう！`
   - 推奨フェーズ: 配信開始直後〜盛り上がり中（ミッション開始前後）
   - 期待CTR帯: 高（実績ベース: 2.34〜3.91%、全タイプ最高CTR）
   - 根拠: サービス紹介フック（7件）のCTR平均2.34%は全タイプ最高。`#SANTACLAWS`（CTR 3.91%）・`#Dear13ro`（CTR 2.57%）・`#AtroceMyrtille`（CTR 3.24%）の3件がいずれも高CTRを記録。ギャング別ファンコミュニティへの精密な訴求と「応援」という行動動詞の組み合わせが機能している。

3. **キャラクター+状況+緊迫感型**
   - 型: `[ギャング名/メンバー名]が[具体的な状況]👀 [行動語]を[マルチビュー等で]見よう！`
   - 推奨フェーズ: 盛り上がり中〜結果共有（ミッション進行中・クライマックス）
   - 期待CTR帯: 中〜高（実績ベース: 1.49〜1.85%）
   - 根拠: 「Dear13roがメイズバンク強盗を始めたみたい」（64 clicks, Imp 3,549）・「飛行場強盗実行中！！ 逃走劇を見よう！！」（57 clicks, Imp 3,222）はともに高インプレッション×高クリックを達成。「みたい」のような推量表現が「情報の確認」への好奇心を刺激し、「逃走劇」のような具体的シーン名が場面の臨場感を補強している。

4. **特別性・限定感付き実況型**
   - 型: `[ギャング名]が[通常と違う理由を一言]の[イベント名]へ🔥 今夜を[マルチビューで]追おう！！`
   - 推奨フェーズ: 盛り上がり中〜結果共有（特別回・節目回の配信）
   - 期待CTR帯: 中（実績ベース: 1.49%、ただし高Imp 2,282で34 clicks）
   - 根拠: 「Dear13roがついにパシフィック銀行へ🔥 今日は一人のメンバーがお休みに入る特別な一戦。師弟13人が揃う今夜をマルチビューで追おう」（URL Clicks: 34, Imp: 2,282）。状況の特殊性がコアファンへの訴求力を高め、通常回と差別化したクリック動機を生む。

5. **集大成・完結フレーム型**
   - 型: `[ギャング名]が最後の[イベント名]を始めた👀 URLからマルチビューを見て[最後の/集大成の][シーン]を見守ろう！！`
   - 推奨フェーズ: 結果共有・節目の配信（最終回・ラストミッション）
   - 期待CTR帯: 高（実績ベース: CTR 4.12%、32 clicks / Imp 776）
   - 根拠: 「Dear13roが最後のメイズ銀行強盗を始めたみたい👀」（URL Clicks: 32, CTR: 4.12%）。Imp 776と低いにもかかわらずCTR 4%超えを達成。「最後」「集大成」という言葉がファンの「絶対に見届けなければ」という義務感に変換されクリック率を押し上げている。節目・最終回フォーマットに特化して使うことで効果を最大化できる。

### 解釈（Phase Analyst + X Algorithm観点）

**[1位実績値] 結果・余韻（avg URL Clicks 64.0・n=1）**

データ上はURL Clicks最多だが、サンプル数が1件のみであり、この投稿（「Dear13roがメイズバンク強盗を始めたみたい」Imp 3,549）は実態として「開始直後」フェーズに近い可能性もある。しかしイベントクライマックス後の余韻が初速を最大化した例外的ケースとして、4投稿フルシーケンスを組む際には「結果・余韻」を意図的に設計することで再現を狙う価値がある。次回以降のサンプル蓄積が必須。

**[実質1位] 開始直後（avg URL Clicks 34.0 / CTR 1.80% / Imp 1,928）**

「今まさに始まった」という時間的切迫感がユーザーの即時クリック行動を最大化する。XアルゴリズムのvelocityシグナルがFor Youへの候補昇格に直接影響するため、イベント開始検知から5分以内の投稿が理想。「開始前告知」はImp 1,760と高いにもかかわらずURL Clicks 6.0・CTR 0.34%に留まり、開始直後との比較で約5.7倍の差がある。告知フェーズはImp確保に貢献するが、主力はあくまで開始直後ポストに置くこと。

**[2位] イベント紹介（avg URL Clicks 25.7 / CTR 1.96%・全フェーズ中最高）**

CTRは全フェーズ中最高（1.96%）。「イベントの文脈と意義を説明した上でURLを提示する」構造が、ユーザーの「なぜ見るべきか」の疑問を解消しクリックを促す。テキストが説明的になりすぎると読まれずにスクロールされるリスクがあるため、最初の1行で強いフックを作りつつ補足情報でCTRを底上げする設計が必要。

**[注意] 実行中・進行中（avg URL Clicks 16.0 / CTR 1.17%）**

投稿数8と最多カテゴリだが平均URL Clicksは開始直後（34.0）の半分以下。「まだ続いている」という情報は緊急性が薄れ、ユーザーが後回しにしやすい。途中参入者の受け皿として「まだ間に合う」「逃走劇を見よう」のような途中参入ハードルを下げるコピーで設計すること。同じフォーマットの繰り返しはエンゲージメントが逓減するため、進行状況（逃走中・成功直前・失敗）などのリアルタイム情報を加えてユニーク性を担保する。

**[最下位] サービス告知（avg URL Clicks 1.0 / CTR 0.38%）**

Impも全フェーズ最低（756）。イベントの文脈から切り離された状態では反応を得にくい。「今日の強盗をマルチビューで見よう」という形でイベントと紐付けるか、フォロワーが多い投稿へのリプライとして露出させる設計が有効。

**拡散型が0件という構造的課題**

現在の35投稿はviral_focused（返信・リポスト誘発型）がゼロ件。クリック特化型8投稿が全URL Clicksの約51%を担う構造であり、どのフェーズ・どのフックで投稿するかの選択が成果の大半を決定している。アルゴリズムのOut-of-Network重みを活かすには、コミュニティ内での話題生成（返信誘発型投稿）の導入が中期課題。

### 推奨投稿シーケンス

**2投稿パターン（最小構成）**

1. **開始直後** — 「[イベント名]が始まった」という即報。URLと短い誘導文のみで完結。 — 配信開始から3〜5分以内
2. **実行中・進行中** — 「まだ間に合う」という途中参入訴求。進行状況（逃走中・佳境など）をテキストで状況共有。 — 開始から30〜60分後

理由: 最小2投稿で「即時ユーザー」と「タイムラインを後から見るユーザー」の両方をカバーできる。開始直後はURL Clicks最大化、2投稿目は取りこぼしの回収。

---

**3投稿パターン（標準構成）**

1. **開始前告知** — 「[時刻]から始まります」という事前アラート。URLは入れずにフォロワーへ予告。 — 配信開始の30〜60分前
2. **開始直後** — 「始まった」即報。URLと「今すぐ見られる」という即時性ワードを入れる。 — 配信開始から3〜5分以内
3. **イベント紹介** — マルチビューで見る意義・配信者の背景を少し加えた誘導投稿。クライマックス直前が理想。 — 佳境・クライマックスのタイミングに合わせて

理由: 事前認知→即時誘導→クライマックス誘導の3段階で、タイムラインを見るタイミングが異なるユーザーをそれぞれカバーする。イベント紹介フェーズのCTR 1.96%を活かしてクライマックス前に投入する。

---

**4投稿パターン（フル活用）**

1. **開始前告知** — 「[時刻]から[イベント名]が始まります」。カウントダウン感を演出。URLなしまたは配信者プロフィールURLのみ。 — 配信開始の60分前
2. **開始直後** — 「[イベント名]が始まった」即報。StreamerHubのURLを前面に出す。フックワードは「始まった」「始まる」形式（avg 34 clicks実績）。 — 配信開始から3〜5分以内
3. **実行中・進行中** — 「まだ間に合う」中間誘導。進行状況をリアルタイムで伝えて続きへの興味を喚起。 — 開始から30〜45分後
4. **結果・余韻（クライマックス報告）** — 「[結果や印象的な出来事]が起きた」という事後報告。次回イベントへの期待を育てる。 — イベント終了直後〜翌日

理由: 4投稿でイベントライフサイクルを完全カバーする。特に4投稿目の「結果・余韻」はデータ上URL Clicks最大（avg 64）を記録しており、イベント終了後も流入を生む効果がある。次回への期待形成にも機能するため、長期的なフォロワー育成を兼ねる。

#### 仮説A: URLをリプライに移動させる「2段階投稿」でインプレッションを増加させ、最終的なURL Clicksの絶対数を伸ばせる

- **仮説**: 本文にURLを含めず「開始直後」フックのみで投稿し、リプライにURLを貼る2段階構成にすることで、本文のImpが現状（avg 1,928）から2,500以上に増加し、リプライのURL Clicksも30以上を達成できる。
- **実験設計の要点**: 同じ日・同じギャングのイベントで、1回はURL本文に含める通常型（A案）、次回は2段階投稿（B案）を試す。リプライURL Clicksの計測方法をあらかじめ設定しておく。計測できない場合はGAの日次X流入数で代替比較する。

#### 仮説B: 「集大成・最後」フレームはギャングの節目イベントに限定して使うことで、節目以外の通常投稿との差別化が維持できる

- **背景**: top_postsのrank10「Dear13roが最後のメイズ銀行強盗を始めたみたい」（32 clicks, CTR 4.12%, Imp 776）はImpが全Top10中最低にもかかわらずCTRが4.12%と最高水準を記録している。「最後」「集大成」という言葉がファンの義務感を引き出していると推察される。節目イベントでない通常回に「最後」を使うと効果が薄れるか逆効果になる。
- **仮説**: 「最後」「集大成」フレームは節目イベントでのみ使うと、CTR 3.5〜4.5%を再現できる。通常投稿に使うと既視感から効果が半減する。
- **実験設計の要点**: 節目イベントか否かを投稿前にメタデータとして記録し、「最後」「集大成」使用の有無と対応するCTR/URL Clicksを蓄積する。10投稿以上のサンプルが集まった段階で比較する。
